# 第一部分：MapGenerator.java —— 地图路径生成器

## 一、类概述

`MapGenerator` 是一个所有方法均为 `static` 的类（但拥有一个公开的无参构造函数 `public MapGenerator() {}`，因此可以被实例化），负责生成地下城的**地图拓扑结构**。它通过创建网格节点、随机生成从起点到终点的多条路径、过滤冗余边，最终产出一个二维的 `MapRoomNode` 网格，其中每个节点可能拥有指向下一行节点的边（即路径连接）。

**核心职责**：生成地图的"骨架"——哪些房间节点之间存在通路。

---

## 二、公开入口方法

### 2.1 `generateDungeon(int height, int width, int pathDensity, Random rng)`

这是整个地图生成的**唯一公开入口**，被外部调用以生成一张完整的地下城地图。

**参数说明**：
| 参数 | 含义 |
|------|------|
| `height` | 地图的行数（即楼层数/深度） |
| `width` | 每行的节点数量（即每层最多可有多少个房间） |
| `pathDensity` | 路径密度，决定从起点出发生成多少条独立路径 |
| `rng` | 随机数生成器实例 |

**执行流程**（严格按代码顺序）：

1. **调用 `createNodes(height, width)`**：创建一个 `height × width` 的二维网格，每个格子是一个空的 `MapRoomNode(x, y)`，此时节点之间没有任何连接（边）。
2. **检查 Mod**：调用 `ModHelper.isModEnabled("Uncertain Future")` 判断是否启用了名为 "Uncertain Future" 的 Mod。
   - 如果**启用**了该 Mod，调用 `createPaths(map, 1, rng)`，将路径密度强制设为 **1**（只生成一条路径）。
   - 如果**未启用**，使用传入的 `pathDensity` 调用 `createPaths(map, pathDensity, rng)`。
3. **调用 `filterRedundantEdgesFromRow(map)`**：对第一行（索引 0）的节点进行冗余边过滤，删除指向同一目标节点的重复边。
4. **返回**处理后的 `map`。

## 三、节点创建

### 3.1 `createNodes(int height, int width)`

创建一个空的二维网格结构。

**逻辑**：
- 外层循环 `y` 从 `0` 到 `height - 1`，逐行创建。
- 内层循环 `x` 从 `0` 到 `width - 1`，逐列创建。
- 每个位置创建一个 `new MapRoomNode(x, y)`，加入当前行的 `ArrayList` 中。
- 每行创建完毕后加入外层的 `ArrayList` 中。

**结果**：返回一个 `ArrayList<ArrayList<MapRoomNode>>`，即一个二维列表，`map.get(y).get(x)` 可以获取坐标为 `(x, y)` 的节点。此时所有节点都是孤立的，没有任何边连接。

## 四、路径生成

### 4.1 `createPaths(ArrayList<ArrayList<MapRoomNode>> nodes, int pathDensity, Random rng)`

这是路径生成的**调度方法**，负责确定每条路径的起始节点，然后委托给 `_createPaths` 进行递归路径构建。

**变量定义**：
- `first_row = 0`：第一行的索引，固定为 0。
- `row_size = nodes.get(first_row).size() - 1`：第一行的最大索引值（即 `width - 1`）。
- `firstStartingNode = -1`：记录第一条路径的起始节点 x 坐标，初始为 -1。

**主循环**（`i` 从 `0` 到 `pathDensity - 1`）：

**步骤一：随机选择起始节点**
```java
int startingNode = randRange(rng, 0, row_size);
```
在 `[0, row_size]` 范围内随机选取一个 x 坐标作为起始节点（由于 `randRange` 是全闭区间，`row_size` 本身也会被选中，实际最大值为 `row_size = width - 1`）。

**步骤二：记录第一条路径的起点**
```java
if (i == 0) {
    firstStartingNode = startingNode;
}
```
当 `i == 0`（即第一条路径）时，将起始节点记录到 `firstStartingNode`。

**步骤三：避免第二条路径与第一条路径起点相同**
```java
while (startingNode == firstStartingNode && i == 1) {
    startingNode = randRange(rng, 0, row_size);
}
```
当 `i == 1`（即第二条路径）时，如果随机到的起点与第一条路径相同，则**重新随机**，直到不同为止。注意：这只对第二条路径（`i == 1`）生效，第三条及之后的路径没有此限制。

**步骤四：递归创建路径**
```java
_createPaths(nodes, new MapEdge(startingNode, -1, startingNode, 0), rng);
```
构造一个虚拟的起始边 `MapEdge(startingNode, -1, startingNode, 0)`，其中：
- `srcX = startingNode`, `srcY = -1`（虚拟源，表示"地图之外"）
- `dstX = startingNode`, `dstY = 0`（目标为第一行的某个节点）

然后调用 `_createPaths` 从这个虚拟边开始递归向下生成路径。

---

### 4.2 `_createPaths(ArrayList<ArrayList<MapRoomNode>> nodes, MapEdge edge, Random rng)` —— 核心递归方法

这是整个地图生成算法的**核心**。它接收一条边，从该边的目标节点出发，向下一行生成新的边，然后递归继续，直到到达地图底部。

#### 4.2.1 获取当前节点

```java
MapRoomNode currentNode = getNode(edge.dstX, edge.dstY, nodes);
```
通过 `edge.dstX` 和 `edge.dstY` 从网格中取出当前节点。

#### 4.2.2 终止条件（到达地图底部）

```java
if (edge.dstY + 1 >= nodes.size()) {
    MapEdge newEdge = new MapEdge(edge.dstX, edge.dstY,
        currentNode.offsetX, currentNode.offsetY,
        3, edge.dstY + 2, 0.0F, 0.0F, true);
    currentNode.addEdge(newEdge);
    Collections.sort(currentNode.getEdges(), new EdgeComparator());
    return nodes;
}
```

当 `edge.dstY + 1 >= nodes.size()` 时，说明当前节点已经在**最后一行**，没有下一行可以连接了。

此时创建一条**特殊边**：
- `dstX = 3`, `dstY = edge.dstY + 2`：目标指向地图之外的固定坐标 `(3, 当前行+2)`，代表通往 Boss 的出口。
- 最后一个参数 `true`：标记这条边为 Boss 边。
- `offsetX`, `offsetY` 使用当前节点的偏移值。
- 权重 `0.0F, 0.0F`。

将这条边添加到当前节点，然后对边列表排序，最后返回。

#### 4.2.3 正常路径生成（非最后一行）

当不在最后一行时，执行以下步骤：

**步骤 A：确定下一行目标节点的 x 坐标范围**

```java
int row_width = nodes.get(edge.dstY).size();
int row_end_node = row_width - 1;
int min, max;
if (edge.dstX == 0) {
    min = 0; max = 1;
} else if (edge.dstX == row_end_node) {
    min = -1; max = 0;
} else {
    min = -1; max = 1;
}
```

根据当前节点的 x 坐标位置，确定下一步的偏移参数 `min` 和 `max`：

| 当前节点位置 | min | max | `randRange(rng, min, max)` 实际返回值 | 实际偏移 |
|-------------|-----|-----|--------------------------------------|---------|
| 最左侧（x == 0） | 0 | 1 | 返回 `0` 或 `1`（全闭区间） | **直行或右移（各50%）** |
| 最右侧（x == row_end_node） | -1 | 0 | 返回 `-1` 或 `0`（全闭区间） | **左移或直行（各50%）** |
| 中间位置 | -1 | 1 | 返回 `-1`、`0` 或 `1`（全闭区间） | **左移、直行或右移（各约33%）** |

> **注意**：`randRange` 是全闭区间 `[min, max]`，`max` 值会被选中。因此中间位置可以三向偏移，最左侧可以右移，最右侧可以直行。

**步骤 B：随机选择初始目标 x 坐标**

```java
int newEdgeX = edge.dstX + randRange(rng, min, max);
int newEdgeY = edge.dstY + 1;
MapRoomNode targetNodeCandidate = getNode(newEdgeX, newEdgeY, nodes);
```

在允许的偏移范围内随机选择一个偏移量，计算出目标节点的坐标。`newEdgeY` 固定为当前行 + 1（即下一行）。

**步骤 C：祖先间距检查（避免路径过于紧密地分叉/汇合）**

```java
int min_ancestor_gap = 3;
int max_ancestor_gap = 5;
ArrayList<MapRoomNode> parents = targetNodeCandidate.getParents();
```

定义两个关键常量：
- `min_ancestor_gap = 3`：最小祖先间距，两条路径至少要间隔 3 行才能重新汇合。
- `max_ancestor_gap = 5`：最大祖先间距，超过 5 行的共同祖先不再考虑。

然后检查目标候选节点是否已经有父节点（即是否已有其他路径连接到它）：

```java
if (!parents.isEmpty()) {
    for (MapRoomNode parent : parents) {
        if (parent != currentNode) {
            MapRoomNode ancestor = getCommonAncestor(parent, currentNode, max_ancestor_gap);
            if (ancestor != null) {
                int ancestor_gap = newEdgeY - ancestor.y;
                if (ancestor_gap < min_ancestor_gap) {
                    // 需要调整 newEdgeX
                } else if (ancestor_gap >= max_ancestor_gap) {
                    // 不做处理（空分支）
                }
            }
        }
    }
}
```

遍历目标候选节点的**所有已存在的父节点**（排除当前节点自身），对每个父节点：
1. 调用 `getCommonAncestor(parent, currentNode, max_ancestor_gap)` 查找该父节点与当前节点的**最近共同祖先**。
2. 如果找到了共同祖先，计算 `ancestor_gap = newEdgeY - ancestor.y`（即从共同祖先到当前行的距离）。
3. **如果 `ancestor_gap < min_ancestor_gap`（间距太小）**：需要调整目标 x 坐标，避免路径过于紧密地汇合。
4. **如果 `ancestor_gap >= max_ancestor_gap`**：不做任何处理（代码中是空分支，即允许这种情况）。

**步骤 C-1：间距过小时的 x 坐标调整策略**

当 `ancestor_gap < min_ancestor_gap` 时，根据目标候选节点与当前节点的相对位置进行不同处理：

**情况 1：目标在当前节点的右侧**（`targetNodeCandidate.x > currentNode.x`）
```java
newEdgeX = edge.dstX + randRange(rng, -1, 0);
if (newEdgeX < 0) {
    newEdgeX = edge.dstX;
}
```
改为向左偏移（返回 -1 或 0，各 50%）。如果调整后 x < 0，则保持不变（`edge.dstX`）。

**情况 2：目标与当前节点同列**（`targetNodeCandidate.x == currentNode.x`）
```java
newEdgeX = edge.dstX + randRange(rng, -1, 1);
if (newEdgeX > row_end_node) {
    newEdgeX = edge.dstX - 1;
} else if (newEdgeX < 0) {
    newEdgeX = edge.dstX + 1;
}
```
随机偏移（返回 -1、0 或 1，各约 33%）。如果越界则强制调整到相邻位置。

**情况 3：目标在当前节点的左侧**（`targetNodeCandidate.x < currentNode.x`）
```java
newEdgeX = edge.dstX + randRange(rng, 0, 1);
if (newEdgeX > row_end_node) {
    newEdgeX = edge.dstX;
}
```
保持不变或向右偏移（返回 0 或 1，各 50%）。如果调整后 x > row_end_node，则保持不变。

调整后重新获取目标节点：
```java
targetNodeCandidate = getNode(newEdgeX, newEdgeY, nodes);
```

**步骤 D：防止边交叉——左邻居检查**

```java
if (edge.dstX != 0) {
    MapRoomNode left_node = nodes.get(edge.dstY).get(edge.dstX - 1);
    if (left_node.hasEdges()) {
        MapEdge right_edge_of_left_node = getMaxEdge(left_node.getEdges());
        if (right_edge_of_left_node.dstX > newEdgeX) {
            newEdgeX = right_edge_of_left_node.dstX;
        }
    }
}
```

当当前节点**不在最左侧**（`edge.dstX != 0`）时：
1. 获取同一行中**左边相邻节点** `left_node`。
2. 如果左邻居已经有边（即已经生成了向下一行的连接）：
3. 取左邻居所有边中 `dstX` **最大**的那条边（即最靠右的连接）。
4. 如果左邻居的最右连接的目标 x **大于**当前计划的目标 x（`right_edge_of_left_node.dstX > newEdgeX`），说明会产生**边交叉**，因此将 `newEdgeX` 调整为 `right_edge_of_left_node.dstX`，使其至少与左邻居的最右连接对齐。

**步骤 E：防止边交叉——右邻居检查**

```java
if (edge.dstX < row_end_node) {
    MapRoomNode right_node = nodes.get(edge.dstY).get(edge.dstX + 1);
    if (right_node.hasEdges()) {
        MapEdge left_edge_of_right_node = getMinEdge(right_node.getEdges());
        if (left_edge_of_right_node.dstX < newEdgeX) {
            newEdgeX = left_edge_of_right_node.dstX;
        }
    }
}
```

当当前节点**不在最右侧**（`edge.dstX < row_end_node`）时：
1. 获取同一行中**右边相邻节点** `right_node`。
2. 如果右邻居已经有边：
3. 取右邻居所有边中 `dstX` **最小**的那条边（即最靠左的连接）。
4. 如果右邻居的最左连接的目标 x **小于**当前计划的目标 x（`left_edge_of_right_node.dstX < newEdgeX`），说明会产生**边交叉**，因此将 `newEdgeX` 调整为 `left_edge_of_right_node.dstX`。

**步骤 F：创建边并递归**

```java
targetNodeCandidate = getNode(newEdgeX, newEdgeY, nodes);
MapEdge newEdge = new MapEdge(edge.dstX, edge.dstY,
    currentNode.offsetX, currentNode.offsetY,
    newEdgeX, newEdgeY,
    targetNodeCandidate.offsetX, targetNodeCandidate.offsetY, false);
currentNode.addEdge(newEdge);
Collections.sort(currentNode.getEdges(), new EdgeComparator());
targetNodeCandidate.addParent(currentNode);
return _createPaths(nodes, newEdge, rng);
```

1. 用最终确定的 `newEdgeX` 和 `newEdgeY` 重新获取目标节点。
2. 创建新的 `MapEdge`，包含源节点坐标、偏移量、目标节点坐标、偏移量，最后一个参数 `false` 表示这不是 Boss 边。
3. 将新边添加到当前节点的边列表中。
4. 对当前节点的边列表按 `EdgeComparator` 排序。
5. 将当前节点添加为目标候选节点的**父节点**（`targetNodeCandidate.addParent(currentNode)`），建立反向引用。
6. **递归调用** `_createPaths(nodes, newEdge, rng)`，以新创建的边为参数继续向下一行生成路径。

## 五、辅助方法

### 5.1 `getMaxEdge(ArrayList<MapEdge> edges)`

对边列表按 `EdgeComparator` 排序后，返回列表中**最后一个**元素（即 `dstX` 最大的边）。如果边列表为空，触发断言错误。

### 5.2 `getMinEdge(ArrayList<MapEdge> edges)`

对边列表按 `EdgeComparator` 排序后，返回列表中**第一个**元素（即 `dstX` 最小的边）。如果边列表为空，触发断言错误。

### 5.3 `getNodeWithMaxX(ArrayList<MapRoomNode> nodes)`

遍历节点列表，返回 `x` 值最大的节点。如果列表为空，触发断言错误。

### 5.4 `getNodeWithMinX(ArrayList<MapRoomNode> nodes)`

遍历节点列表，返回 `x` 值最小的节点。如果列表为空，触发断言错误。

### 5.5 `getCommonAncestor(MapRoomNode node1, MapRoomNode node2, int max_depth)`

查找两个节点的**最近共同祖先**。

**前置断言**：
- `node1.y == node2.y`：两个节点必须在同一行。
- `node1 != node2`：两个节点不能是同一个节点。

**逻辑**：
1. 根据 x 坐标确定左右节点：x 较小的为 `l_node`（左节点），x 较大的为 `r_node`（右节点）。
   - 注意：代码中有一个**Bug**——比较条件是 `node1.x < node2.y`（第 148 行），而 `node2.y` 应该是 `node2.x`。
   - **Bug 的精确影响**：由于前置断言 `node1.y == node2.y`（两个节点在同一行），所以 `node2.y` 的值等于 `node1.y`。因此 `node1.x < node2.y` 实际比较的是 `node1.x < node1.y`（即当前节点的 x 坐标是否小于其 y 坐标），**而不是** `node1.x < node2.x`（两个节点谁的 x 更小）。
   - **具体场景分析**：
     - 当 `node1.x < node2.y`（由于 `node2.y == node1.y`，即 `node1.x < node1.y`）时：`l_node = node1, r_node = node2`。这与正确逻辑 `node1.x < node2.x` 的结果**一致**（前提是 `node1.x < node2.x`）。
     - 当 `node1.x >= node2.y`（即 `node1.x >= node1.y`）时：`l_node = node2, r_node = node1`。这与正确逻辑**不一致**——即使 `node1.x < node2.x`，只要 `node1.x >= node1.y`，就会错误地交换左右节点。
     - **实际影响范围**：在标准地图中，`y` 从 0 到 14，`x` 从 0 到 6。当 `y` 较大（如 `y >= 7`）时，几乎所有 `x` 值都满足 `x < y`，Bug 不触发。当 `y` 较小（如 `y = 0, 1, 2`）且 `x >= y` 时，Bug 可能导致左右节点被错误交换，从而影响共同祖先的查找结果。
   - **复刻建议**：如果要精确复刻原始行为，需要保留这个 Bug（使用 `node2.y` 而非 `node2.x`）。如果要修复，则改为 `node2.x`。
2. 从当前行开始，向上逐行回溯：
   - 循环条件：`current_y >= 0` 且 `current_y >= node1.y - max_depth`（不超过最大回溯深度）。
   - 每次迭代：
     - 取 `l_node` 的所有父节点中 `x` **最大**的那个（`getNodeWithMaxX`）。
     - 取 `r_node` 的所有父节点中 `x` **最小**的那个（`getNodeWithMinX`）。
     - 如果两者是同一个节点（`l_node == r_node`），则找到了共同祖先，返回它。
     - 否则 `current_y--`，继续向上回溯。
   - 如果任一节点没有父节点，返回 `null`（未找到共同祖先）。
   - 如果超出最大深度，返回 `null`。

**核心思想**：左节点取其父节点中最靠右的，右节点取其父节点中最靠左的，如果它们相遇了，就找到了分叉点。

### 5.6 `getNode(int x, int y, ArrayList<ArrayList<MapRoomNode>> nodes)`

简单的坐标查找：`nodes.get(y).get(x)`。

### 5.7 `randRange(Random rng, int min, int max)`

生成 `[min, max]` 范围内的随机整数（全闭区间）。

```java
return rng.random(max - min) + min;
```

注意：`rng.random(n)` 内部调用 `random.nextInt(n + 1)`，返回 `[0, n]` 范围的值（全闭区间），所以 `rng.random(max - min)` 返回 `[0, max-min]`，加上 `min` 后得到 `[min, max]`。**`max` 值会被包含在内**（全闭区间）。

**安全检查**：如果 `rng == null`，记录错误日志并用 `new Random(1L)` 作为后备。

## 六、冗余边过滤

### 6.1 `filterRedundantEdgesFromRow(ArrayList<ArrayList<MapRoomNode>> map)`

仅对**第一行**（索引 0）的节点进行冗余边过滤。

**逻辑**：
1. 创建两个列表：`existingEdges`（已存在的边）和 `deleteList`（待删除的边）。
2. 遍历第一行的每个节点：
   - 如果该节点有边：
     - 遍历该节点的每条边：
       - 与 `existingEdges` 中的每条已有边比较，如果 `dstX` 和 `dstY` 都相同（即指向同一个目标节点），则将当前边加入 `deleteList`。
       - 比较完毕后，将当前边加入 `existingEdges`。
     - 遍历 `deleteList`，调用 `node.delEdge(edge)` 删除重复边。
     - 清空 `deleteList`。

**目的**：由于多条路径可能从第一行的不同节点出发，但某些路径可能恰好连接到第一行的同一个节点，导致该节点出现指向同一目标的重复边。此方法确保每个目标节点只被连接一次。

### 7.1 `toString(ArrayList<ArrayList<MapRoomNode>> nodes)` / `toString(nodes, showRoomSymbols)`

**逻辑**：
- 从最后一行（`row_num = nodes.size() - 1`）开始，向上逐行输出（即 Boss 在上，起点在下）。
- 每行输出两部分：
  1. **连接符号行**：显示每个节点向下的连接方向：
     - `\` 表示向左下连接（`edge.dstX < node.x`）
     - `|` 表示正下方连接（`edge.dstX == node.x`）

  2. **节点符号行**：显示每个节点的房间符号：
     - 非最后一行：只有有边的节点才显示符号（通过 `node.getRoomSymbol(showRoomSymbols)` 获取）。
     - 最后一行（起点行）：检查下一行中是否有边指向该节点，如果有则显示符号。

# 第二部分：RoomTypeAssigner.java —— 房间类型分配器

## 一、类概述

`RoomTypeAssigner` 是一个所有方法均为 `static` 的类（同样拥有公开的无参构造函数 `public RoomTypeAssigner() {}`），负责在地图拓扑结构（由 `MapGenerator` 生成）之上，为每个有连接的节点**分配具体的房间类型**（如怪物房、精英房、休息房、商店、事件房、宝箱房等）。

**核心职责**：将房间类型列表按照规则分配到地图的各个节点上，确保不违反各种约束条件。

---

## 二、公开入口方法

### 2.1 `distributeRoomsAcrossMap(Random rng, ArrayList<ArrayList<MapRoomNode>> map, ArrayList<AbstractRoom> roomList)`

这是房间分配的**唯一公开入口**。

**参数说明**：
| 参数 | 含义 |
|------|------|
| `rng` | 随机数生成器 |
| `map` | 由 MapGenerator 生成的地图网格 |
| `roomList` | 待分配的房间类型列表（包含各种 AbstractRoom 子类实例） |

**执行流程**：

**步骤一：统计需要分配房间的节点数量**
```java
int nodeCount = getConnectedNonAssignedNodeCount(map);
```
调用辅助方法统计地图中所有**有边且尚未分配房间**的节点数量。

**步骤二：确保房间数量足够**
```java
while (roomList.size() < nodeCount) {
    roomList.add(new MonsterRoom());
}
```
如果房间列表中的房间数量少于需要分配的节点数量，则不断添加 `MonsterRoom`（普通怪物房）直到数量足够。**MonsterRoom 是默认的填充房间类型**。

**步骤三：处理房间数量过多的情况**
```java
if (roomList.size() > nodeCount) {
    logger.info("WARNING: the roomList is larger than the number of connected nodes. Not all desired roomTypes will be used.");
}
```
如果房间数量多于节点数量，仅记录一条警告日志，不做其他处理（多余的房间不会被使用）。

**步骤四：随机打乱房间列表**
```java
Collections.shuffle(roomList, rng.random);
```
使用随机数生成器对房间列表进行洗牌，确保分配顺序的随机性。

**步骤五：执行房间分配**
```java
assignRoomsToNodes(map, roomList);
```
调用核心分配方法。

**步骤六：记录未分配的房间**
```java
logger.info("#### Unassigned Rooms:");
for (AbstractRoom r : roomList) {
    logger.info(r.getClass());
}
```
分配完成后，`roomList` 中剩余的房间就是未被使用的，记录到日志中。

**步骤七：最终检查**
```java
lastMinuteNodeChecker(map, null);
```
调用最终检查方法，确保没有遗漏的未分配节点。

**步骤八：返回地图**
```java
return map;
```

## 三、核心分配逻辑

### 3.1 `assignRoomsToNodes(ArrayList<ArrayList<MapRoomNode>> map, ArrayList<AbstractRoom> roomList)`

遍历地图中的所有节点，为每个符合条件的节点分配房间。

**逻辑**：
```java
for (ArrayList<MapRoomNode> row : map) {
    for (MapRoomNode node : row) {
        if (node != null && node.hasEdges() && node.getRoom() == null) {
            AbstractRoom roomToBeSet = getNextRoomTypeAccordingToRules(map, node, roomList);
            if (roomToBeSet != null) {
                node.setRoom((AbstractRoom) roomList.remove(roomList.indexOf(roomToBeSet)));
            }
        }
    }
}
```

遍历顺序：**从上到下、从左到右**（外层循环行，内层循环列）。

对每个节点，满足以下**全部条件**才进行分配：
1. `node != null`：节点不为空。
2. `node.hasEdges()`：节点有边连接（即该节点在路径上）。
3. `node.getRoom() == null`：节点尚未被分配房间。

分配过程：
1. 调用 `getNextRoomTypeAccordingToRules(map, node, roomList)` 获取一个符合规则的房间类型。
2. 如果返回了有效的房间（不为 `null`），则：
   - 从 `roomList` 中**移除**该房间实例（`roomList.remove(roomList.indexOf(roomToBeSet))`）。
   - 将该房间设置到节点上（`node.setRoom(...)`）。

**关键点**：房间是从 `roomList` 中移除后使用的，所以每个房间实例只能被分配一次。

---

### 3.2 `getNextRoomTypeAccordingToRules(ArrayList<ArrayList<MapRoomNode>> map, MapRoomNode n, ArrayList<AbstractRoom> roomList)`

根据规则从房间列表中找到第一个适合分配给节点 `n` 的房间类型。

**逻辑**：
```java
ArrayList<MapRoomNode> parents = n.getParents();
ArrayList<MapRoomNode> siblings = getSiblings(map, parents, n);

for (AbstractRoom roomToBeSet : roomList) {
    if (ruleAssignableToRow(n, roomToBeSet)) {
        if (!ruleParentMatches(parents, roomToBeSet) && !ruleSiblingMatches(siblings, roomToBeSet)) {
            return roomToBeSet;
        }
        if (n.y == 0) {
            return roomToBeSet;
        }
    }
}
return null;
```

1. 获取当前节点的**父节点列表**和**兄弟节点列表**。
2. 遍历 `roomList` 中的每个房间：
   - **首先检查** `ruleAssignableToRow`：该房间类型是否允许出现在当前行。
   - **如果允许出现在当前行**：
     - **如果父节点和兄弟节点都没有相同类型的房间**（`!ruleParentMatches && !ruleSiblingMatches`），则返回该房间。
     - **特殊例外**：如果当前节点在**第 0 行**（`n.y == 0`），则**无视父节点和兄弟节点的匹配规则**，直接返回该房间。这意味着第一行的节点可以接受任何允许在该行的房间类型，即使与邻居重复。
3. 如果遍历完所有房间都没有找到合适的，返回 `null`（该节点暂时不会被分配房间）。

## 四、三条分配规则

### 4.1 `ruleAssignableToRow(MapRoomNode n, AbstractRoom roomToBeSet)` —— 行限制规则

判断某种房间类型是否允许被分配到节点 `n` 所在的行。

**适用房间类型及限制**：

```java
List<Class<? extends AbstractRoom>> applicableRooms = Arrays.asList(RestRoom.class, MonsterRoomElite.class);
List<Class<RestRoom>> applicableRooms2 = Collections.singletonList(RestRoom.class);
```

**规则 1**：`RestRoom`（休息房）和 `MonsterRoomElite`（精英房）**不能出现在前 5 行**（`n.y <= 4`）。
```java
if (n.y <= 4 && applicableRooms.contains(roomToBeSet.getClass())) {
    return false;
}
```

**规则 2**：`RestRoom`（休息房）**不能出现在第 13 行及之后**（`n.y >= 13`）。
```java
return n.y < 13 || !applicableRooms2.contains(roomToBeSet.getClass());
```

**总结**：

| 房间类型 | 限制 |
|---------|------|
| `RestRoom`（休息房） | 不能在第 0~4 行，也不能在第 13 行及之后 |
| `MonsterRoomElite`（精英房） | 不能在第 0~4 行 |
| 其他房间类型 | 无行限制 |

### 4.2 `ruleParentMatches(ArrayList<MapRoomNode> parents, AbstractRoom roomToBeSet)` —— 父节点重复规则

判断待分配房间的类型是否与某个父节点的房间类型相同。

**适用房间类型**：
```java
List<Class<? extends AbstractRoom>> applicableRooms = Arrays.asList(
    RestRoom.class, TreasureRoom.class, ShopRoom.class, MonsterRoomElite.class
);
```

只有以下四种房间类型会触发此规则：
- `RestRoom`（休息房）
- `TreasureRoom`（宝箱房）
- `ShopRoom`（商店）
- `MonsterRoomElite`（精英房）

**逻辑**：
```java
for (MapRoomNode parentNode : parents) {
    AbstractRoom parentRoom = parentNode.getRoom();
    if (parentRoom != null && applicableRooms.contains(roomToBeSet.getClass())
        && roomToBeSet.getClass().equals(parentRoom.getClass())) {
        return true;
    }
}
return false;
```

遍历所有父节点：
- 如果父节点已经被分配了房间，且待分配房间的类型在适用列表中，且**与父节点的房间类型完全相同**，则返回 `true`（表示匹配/冲突）。
- 否则返回 `false`。

**含义**：休息房、宝箱房、商店、精英房**不能与直接父节点的类型相同**（避免连续出现同类特殊房间）。注意 `MonsterRoom`（普通怪物房）和 `EventRoom`（事件房）**不受此规则限制**。

### 4.3 `ruleSiblingMatches(ArrayList<MapRoomNode> siblings, AbstractRoom roomToBeSet)` —— 兄弟节点重复规则

判断待分配房间的类型是否与某个兄弟节点的房间类型相同。

**适用房间类型**：
```java
List<Class<? extends AbstractRoom>> applicableRooms = Arrays.asList(
    RestRoom.class, MonsterRoom.class, EventRoom.class, MonsterRoomElite.class, ShopRoom.class
);
```

以下五种房间类型会触发此规则：
- `RestRoom`（休息房）
- `MonsterRoom`（普通怪物房）
- `EventRoom`（事件房）
- `MonsterRoomElite`（精英房）
- `ShopRoom`（商店）

注意：与父节点规则不同，这里多了 `MonsterRoom` 和 `EventRoom`，但少了 `TreasureRoom`。

**逻辑**：
```java
for (MapRoomNode siblingNode : siblings) {
    if (siblingNode.getRoom() != null && applicableRooms.contains(roomToBeSet.getClass())
        && roomToBeSet.getClass().equals(siblingNode.getRoom().getClass())) {
        return true;
    }
}
return false;
```

遍历所有兄弟节点：
- 如果兄弟节点已经被分配了房间，且待分配房间的类型在适用列表中，且**与兄弟节点的房间类型完全相同**，则返回 `true`。
- 否则返回 `false`。

**含义**：休息房、怪物房、事件房、精英房、商店**不能与同一层的兄弟节点类型相同**（避免同一层出现重复的特殊房间）。注意 `TreasureRoom`（宝箱房）**不受此规则限制**。

## 五、辅助方法

### 5.1 `getConnectedNonAssignedNodeCount(List<ArrayList<MapRoomNode>> map)`

统计地图中所有满足 `hasEdges() == true` 且 `getRoom() == null` 的节点数量。即：有路径连接但尚未被分配房间的节点总数。

### 5.2 `getSiblings(List<ArrayList<MapRoomNode>> map, ArrayList<MapRoomNode> parents, MapRoomNode n)`

获取节点 `n` 的所有**兄弟节点**。

**兄弟节点的定义**：与节点 `n` 拥有**共同父节点**的其他节点。

**逻辑**：
```java
for (MapRoomNode parent : parents) {
    for (MapEdge parentEdge : parent.getEdges()) {
        MapRoomNode siblingNode = map.get(parentEdge.dstY).get(parentEdge.dstX);
        if (!siblingNode.equals(n)) {
            siblings.add(siblingNode);
        }
    }
}
```

遍历当前节点的每个父节点，再遍历每个父节点的所有边（即父节点的所有子节点），排除当前节点自身，其余都是兄弟节点。

**注意**：同一个兄弟节点可能被添加多次（如果两个父节点都连接到同一个子节点），列表中可能存在重复。

### 5.3 `lastMinuteNodeChecker(ArrayList<ArrayList<MapRoomNode>> map, MapRoomNode n)`

最终安全检查，确保没有遗漏的未分配节点。

**逻辑**：
```java
for (ArrayList<MapRoomNode> row : map) {
    for (MapRoomNode node : row) {
        if (node != null && node.hasEdges() && node.getRoom() == null) {
            logger.info("INFO: Node=" + node.toString() + " was null. Changed to a MonsterRoom.");
            node.setRoom(new MonsterRoom());
        }
    }
}
```

遍历地图中的所有节点，如果有任何节点满足：
- 不为空
- 有边连接
- 房间为空

则将其强制设为 `MonsterRoom`（普通怪物房），并记录日志。这是一个**兜底机制**，确保所有路径上的节点都有房间类型。

注意：参数 `MapRoomNode n` 在方法体中**完全没有被使用**，传入的值始终为 `null`。

### 5.4 `assignRowAsRoomType(ArrayList<MapRoomNode> row, Class<? extends AbstractRoom> c)`

将一整行的所有未分配节点统一设为指定的房间类型。

**逻辑**：
```java
for (MapRoomNode n : row) {
    if (n.getRoom() == null) {
        try {
            n.setRoom((AbstractRoom) c.newInstance());
        } catch (IllegalAccessException | InstantiationException e) {
            ((ReflectiveOperationException) e).printStackTrace();
        }
    }
}
```

遍历行中的每个节点，如果房间为空，则通过反射创建指定类的实例并设置。此方法在 `distributeRoomsAcrossMap` 的主流程中**未被调用**，可能被其他外部代码使用（例如强制设置某一行为特定类型）。

## 九、概率与随机性完全解析（复刻关键）

> 本节专门针对复刻需求，逐一拆解代码中**每一处随机调用**的精确行为，包括实际概率分布、区间边界、以及容易被忽略的细节。

### 9.1 `randRange` 的精确语义（复刻最关键的函数）

```java
private static int randRange(Random rng, int min, int max) {
    if (rng == null) {
        logger.info("RNG WAS NULL, REPORT IMMEDIATELY");
        rng = new Random(1L);
    }
    return rng.random(max - min) + min;
}
```

**`com.megacrit.cardcrawl.random.Random` 的 `random(int range)` 实现**（来自用户提供的源码）：
```java
public int random(int range) {
    ++this.counter;
    return this.random.nextInt(range + 1);  // this.random 类型为 libGDX 的 RandomXS128
}
```

> **说明**：`RandomXS128.nextInt(n)` 是 libGDX 库的实现，其行为（返回 `[0, n)` 范围的整数）基于 libGDX 的公开文档和通用实现。此行为未在本次分析的源码文件中直接验证，复刻时需自行确认。

**精确行为**：
- `random.nextInt(n)` 返回 `[0, n)` 范围的整数（左闭右开）
- 因此 `rng.random(range)` 返回 `[0, range]` 范围的整数（全闭区间）
- 因此 `randRange(rng, min, max)` 返回的是 **`[min, max]`** 范围的整数
- **`max` 值会被包含在内**

**复刻要点**：如果你的随机函数是 `[min, max)`（左闭右开），则需要传入 `max - 1` 才能等价。例如：
- 代码中 `randRange(rng, -1, 1)` 实际返回 `-1`、`0` 或 `1`（**包含 1**）
- 等价于 `[min, max)` 区间的 `randRange(rng, -1, 2)`

**各调用点的实际取值范围一览**：

| 调用位置 | 代码中的调用 | 实际返回值 |
|---------|------------|-----------|
| 起始节点选择 | `randRange(rng, 0, row_size)` | `[0, row_size]` 即 `0` 到 `width-1` |
| 中间位置偏移 | `randRange(rng, -1, 1)` | `-1`、`0` 或 `1`（**包含 1**） |
| 最左侧偏移 | `randRange(rng, 0, 1)` | `0` 或 `1`（**包含 1**） |
| 最右侧偏移 | `randRange(rng, -1, 0)` | `-1` 或 `0`（**包含 0**） |
| 祖先检查-情况1 | `randRange(rng, -1, 0)` | `-1` 或 `0` |
| 祖先检查-情况2 | `randRange(rng, -1, 1)` | `-1`、`0` 或 `1` |
| 祖先检查-情况3 | `randRange(rng, 0, 1)` | `0` 或 `1` |

### 9.2 路径起始节点的概率分布

```java
int startingNode = randRange(rng, 0, row_size);
// row_size = width - 1
```

**实际范围**：`[0, width - 1]`，即 `0` 到 `width - 1`。

**所有节点都可能被选为起始点**。假设 `width = 7`（标准地图宽度），`row_size = 6`，则起始节点在 `x ∈ {0, 1, 2, 3, 4, 5, 6}` 中均匀随机选取。

**每条路径的起始点概率**（以 `width = 7` 为例）：
| x 坐标 | 概率 |
|--------|------|
| 0 | 1/7 |
| 1 | 1/7 |
| 2 | 1/7 |
| 3 | 1/7 |
| 4 | 1/7 |
| 5 | 1/7 |
| 6 | 1/7 |

**第二条路径的去重机制**：
```java
while (startingNode == firstStartingNode && i == 1) {
    startingNode = randRange(rng, 0, row_size);
}
```
- 仅对 `i == 1`（第二条路径）生效
- 第二条路径的起始点被**条件拒绝采样**：如果与第一条相同则重抽
- 实际概率：第二条路径的起始点在剩余 `width - 1` 个值中均匀分布
- **第三条及之后的路径没有去重**，可能与之前的路径起点相同

### 9.3 每一步路径偏移的概率分布

#### 9.3.1 初始偏移选择

**中间位置**（`0 < edge.dstX < row_end_node`）：
```java
min = -1; max = 1;
int newEdgeX = edge.dstX + randRange(rng, -1, 1);
```
`randRange(rng, -1, 1)` 返回 `-1`、`0` 或 `1`（全闭区间）。

| 偏移量 | 概率 |
|-------|------|
| -1（左移） | 约 33.3% |
| 0（直行） | 约 33.3% |
| +1（右移） | 约 33.3% |

**最左侧位置**（`edge.dstX == 0`）：
```java
min = 0; max = 1;
int newEdgeX = edge.dstX + randRange(rng, 0, 1);
```
`randRange(rng, 0, 1)` 返回 `0` 或 `1`。

| 偏移量 | 概率 |
|-------|------|
| 0（直行） | 50% |
| +1（右移） | 50% |

**最右侧位置**（`edge.dstX == row_end_node`）：
```java
min = -1; max = 0;
int newEdgeX = edge.dstX + randRange(rng, -1, 0);
```
`randRange(rng, -1, 0)` 返回 `-1` 或 `0`。

| 偏移量 | 概率 |
|-------|------|
| -1（左移） | 50% |
| 0（直行） | 50% |

**初始偏移总结**：

| 当前位置 | 左移(-1) | 直行(0) | 右移(+1) |
|---------|---------|--------|---------|
| 最左侧(x=0) | 0% | 50% | 50% |
| 中间位置 | 约 33% | 约 33% | 约 33% |
| 最右侧(x=row_end) | 50% | 50% | 0% |

**这意味着**：路径可以向三个方向偏移。中间位置三等分，最左侧可以直行或右移，最右侧可以左移或直行。

---

#### 9.3.2 祖先间距检查触发时的偏移调整

当 `ancestor_gap < min_ancestor_gap`（间距 < 3）时，根据目标候选节点与当前节点的相对位置重新随机选择偏移：

**情况 1：目标在右侧**（`targetNodeCandidate.x > currentNode.x`）
```java
newEdgeX = edge.dstX + randRange(rng, -1, 0);
```
`randRange(rng, -1, 0)` 返回 `-1` 或 `0`。

| 调整后偏移 | 概率 |
|-----------|------|
| -1（左移） | 50% |
| 0（直行） | 50% |

边界修正：如果 `newEdgeX < 0`，则 `newEdgeX = edge.dstX`（保持原位）。

**情况 2：目标在同列**（`targetNodeCandidate.x == currentNode.x`）
```java
newEdgeX = edge.dstX + randRange(rng, -1, 1);
```
`randRange(rng, -1, 1)` 返回 `-1`、`0` 或 `1`。

| 调整后偏移 | 概率 |
|-----------|------|
| -1（左移） | 约 33% |
| 0（直行） | 约 33% |
| +1（右移） | 约 33% |

边界修正：
- 如果 `newEdgeX > row_end_node`，则 `newEdgeX = edge.dstX - 1`（强制左移）
- 如果 `newEdgeX < 0`，则 `newEdgeX = edge.dstX + 1`（强制右移）

**情况 3：目标在左侧**（`targetNodeCandidate.x < currentNode.x`）
```java
newEdgeX = edge.dstX + randRange(rng, 0, 1);
```
`randRange(rng, 0, 1)` 返回 `0` 或 `1`。

| 调整后偏移 | 概率 |
|-----------|------|
| 0（直行） | 50% |
| +1（右移） | 50% |

边界修正：如果 `newEdgeX > row_end_node`，则 `newEdgeX = edge.dstX`（保持原位）。

**祖先检查调整总结**（仅 `randRange` 随机部分的概率，不含边界修正）：

| 触发条件 | 随机部分调整方向 | 概率 | 边界修正可能产生的额外方向 |
|---------|---------------|------|--------------------------|
| 目标在右侧 | 左移(-1) | 50% | 无（`newEdgeX < 0` 时保持原位） |
| 目标在右侧 | 直行(0) | 50% | 无 |
| 目标在同列 | 左移(-1) | 约 33% | 无 |
| 目标在同列 | 直行(0) | 约 33% | `newEdgeX < 0` 时**右移(+1)** |
| 目标在同列 | 右移(+1) | 约 33% | `newEdgeX > row_end_node` 时**左移(-1)** |
| 目标在左侧 | 直行(0) | 50% | 无 |
| 目标在左侧 | 右移(+1) | 50% | `newEdgeX > row_end_node` 时保持原位 |

> **注意**：边界修正产生的方向改变仅在越界时触发。

### 9.4 祖先间距检查的触发条件与概率影响

祖先检查**仅在目标候选节点已有其他父节点时**才会触发：
```java
ArrayList<MapRoomNode> parents = targetNodeCandidate.getParents();
if (!parents.isEmpty()) { ... }
```

**触发条件链**：
1. 随机选择了一个目标节点
2. 该目标节点已经有至少一个父节点（即其他路径已经连接到了它）
3. 在这些父节点中，存在一个不是当前节点的父节点
4. 该父节点与当前节点在 `max_ancestor_gap = 5` 行内存在共同祖先
5. 该共同祖先距当前行的距离 `ancestor_gap < min_ancestor_gap = 3`

**只有以上全部满足时**，才会触发偏移调整。否则保持初始随机结果。

**三个区间的行为差异**：

| ancestor_gap 值 | 行为 |
|----------------|------|
| `< 3`（太近） | 触发偏移调整，重新随机选择方向 |
| `3 ~ 4`（中间值） | 不触发调整，保持初始随机结果 |
| `>= 5` | 代码中有空分支 `else if (ancestor_gap >= max_ancestor_gap) {}`，不做任何处理 |

**重要**：当 `ancestor_gap` 恰好为 3 或 4 时，既不满足 `< 3` 也不满足 `>= 5`，所以**不做任何处理**，保持原始的 `newEdgeX`。

---

### 9.5 邻居交叉检查的确定性修正

**左邻居检查**和**右邻居检查**是**确定性**的（不涉及随机），但它们会覆盖之前随机选择的结果：

```java
// 左邻居检查
if (right_edge_of_left_node.dstX > newEdgeX) {
    newEdgeX = right_edge_of_left_node.dstX;  // 强制对齐
}

// 右邻居检查
if (left_edge_of_right_node.dstX < newEdgeX) {
    newEdgeX = left_edge_of_right_node.dstX;  // 强制对齐
}
```

**复刻要点**：这两个检查是**顺序执行**的，不是互斥的。先执行左邻居检查，再执行右邻居检查，后者可能覆盖前者的结果。

**触发条件**：只有当邻居节点**已经有边**（`hasEdges()`）时才检查。由于路径是按顺序生成的（第一条路径完全生成后才开始第二条），**先生成的路径不受后生成路径的约束，但后生成的路径受先生成路径的约束**。

---

### 9.6 多路径生成的顺序依赖性

路径是**串行生成**的：完全生成第一条路径后，才开始生成第二条。

```java
for (int i = 0; i < pathDensity; ++i) {
    // ...选择起始点...
    _createPaths(nodes, new MapEdge(startingNode, -1, startingNode, 0), rng);
}
```

**对概率的影响**：
1. **第一条路径**：不受任何约束，完全自由随机（除了边界限制和祖先检查）
2. **第二条路径**：
   - 起始点不能与第一条相同（条件拒绝采样）
   - 在递归过程中，祖先检查可能检测到与第一条路径的共同祖先
   - 邻居交叉检查会受到第一条路径已生成边的影响
3. **第三条及之后**：
   - 起始点可能与之前的路径相同（没有去重）
   - 受所有已生成路径的约束

**这意味着**：`pathDensity` 的值越大，后面的路径受到的约束越多，随机性越低，路径形状越"被迫"。

---

### 9.7 `filterRedundantEdgesFromRow` 的概率无关性

冗余边过滤是**确定性**操作，不涉及任何随机调用。它只是简单地删除第一行中指向同一目标的重复边。但它的存在意味着：**即使两条路径在第一行选择了不同的起始点，它们后续可能在第一行产生指向同一目标的边，这些重复边会被删除**。

**对复刻的影响**：这个过滤只作用于第一行，且是后处理步骤。在复刻时，可以先不实现它，最后统一过滤即可。

### 9.8 RoomTypeAssigner 中的概率分析

#### 9.8.1 房间列表洗牌

```java
Collections.shuffle(roomList, rng.random);
```

使用 `rng.random`（Java 的 `Random` 实例）作为随机源对房间列表进行 Fisher-Yates 洗牌。**所有房间类型的排列顺序是等概率的**。

#### 9.8.2 房间分配的"先到先得"机制

房间分配的核心逻辑是**遍历打乱后的列表，取第一个通过规则的房间**：

```java
for (AbstractRoom roomToBeSet : roomList) {
    if (ruleAssignableToRow(n, roomToBeSet)) {
        if (!ruleParentMatches(parents, roomToBeSet) && !ruleSiblingMatches(siblings, roomToBeSet)) {
            return roomToBeSet;
        }
        if (n.y == 0) {
            return roomToBeSet;
        }
    }
}
```

**这不是均匀随机分配**！实际概率取决于：
1. **洗牌后的顺序**：列表中排在前面的房间有更高的概率被选中
2. **规则过滤**：某些房间类型可能因为规则被跳过
3. **遍历顺序**：节点从上到下、从左到右处理，先处理的节点有更多房间可选
4. **房间消耗**：被选中的房间从列表中移除，后续节点不能再选

**实际效果**：
- 列表中靠前的房间类型更容易被分配到地图上方的节点
- 列表中靠后的房间类型更容易被分配到地图下方的节点（如果还没被消耗完）
- 如果某种房间类型因为规则限制无法分配到某个节点，它会"跳过"该节点，可能在后面的节点被分配

#### 9.8.3 第 0 行的特殊概率

```java
if (n.y == 0) {
    return roomToBeSet;
}
```

第 0 行的节点**无视父节点和兄弟节点的重复规则**。只要该房间类型允许出现在第 0 行（通过 `ruleAssignableToRow`），就直接返回。这意味着第 0 行的节点总是得到列表中**第一个通过行限制的房间**。

#### 9.8.4 MonsterRoom 的兜底概率

```java
while (roomList.size() < nodeCount) {
    roomList.add(new MonsterRoom());
}
```

当房间列表不足时，用 `MonsterRoom` 填充。**MonsterRoom 不受父节点重复规则和行限制规则的约束**（不在任何 `applicableRooms` 列表中），所以它几乎可以被分配到任何节点。

**实际效果**：如果特殊房间（精英、商店、休息等）因为规则限制无法分配到某些节点，这些节点最终会被 `MonsterRoom` 填充。再加上 `lastMinuteNodeChecker` 的兜底机制，**地图上不可能出现没有房间类型的节点**。

#### 9.8.5 `lastMinuteNodeChecker` 的确定性兜底

```java
node.setRoom(new MonsterRoom());
```

这是纯确定性操作，不涉及随机。所有未被分配的节点统一设为 `MonsterRoom`。

### 9.9 完整概率流程图（单步路径生成）

```
_createPaths 单步执行的概率流程：

1. 确定偏移范围 [min, max]
   │
   ├── x == 0        → min=0, max=1  → randRange 返回 {0, 1}，各50%
   ├── x == row_end  → min=-1, max=0 → randRange 返回 {-1, 0}，各50%
   └── 中间位置       → min=-1, max=1 → randRange 返回 {-1, 0, 1}，各约33%
   │
2. 初始随机选择 newEdgeX
   │
3. 目标节点是否已有其他父节点？
   ├── 否 → 跳过祖先检查
   └── 是 → 遍历每个其他父节点：
         │
         ├── 在5行内找到共同祖先？
         │   ├── 否 → 跳过
         │   └── 是 → ancestor_gap < 3？
         │       ├── 否（3或4）→ 不调整
         │       └── 是（0、1、2）→ 根据相对位置重新随机：
         │           ├── 目标在右 → 50%左移, 50%直行
         │           ├── 目标同列 → 约33%左移, 约33%直行, 约33%右移
         │           │   └── 边界修正：越界时强制调整
         │           └── 目标在左 → 50%直行, 50%右移
         │
4. 左邻居交叉检查（确定性覆盖）
   │
5. 右邻居交叉检查（确定性覆盖）
   │
6. 最终确定 newEdgeX，创建边，递归
```

## 十、两个类的协作关系

```
MapGenerator.generateDungeon()
    → 生成地图拓扑结构（哪些节点之间有路径连接）
    → 返回 ArrayList<ArrayList<MapRoomNode>>

RoomTypeAssigner.distributeRoomsAcrossMap()
    → 接收 MapGenerator 的输出
    → 为每个路径上的节点分配具体的房间类型
    → 返回最终的完整地图
```

**MapGenerator** 解决的是"**哪里有路**"的问题，**RoomTypeAssigner** 解决的是"**路上是什么房间**"的问题。两者组合构成了完整的地图生成管线。
